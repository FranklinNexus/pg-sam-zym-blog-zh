# Python 悖论

> 原文：<https://paulgraham.com/pypar.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：The Python Paradox
> 年代：2004 年 8 月

### Python 悖论

在一次演讲里，我说了一句惹恼不少人的话：做 Python 项目，你能招到比 Java 项目**更聪明**的程序员。

我不是说 Java 程序员笨，而是说 **Python 程序员更聪明**。学一门新语言很累；人们学 Python，多半不是因为好找工作，而是真的爱编程，且对已有语言不满意。

这正是公司该雇的人。于是有——姑且叫它——**Python 悖论**：公司若选相对生僻的语言写软件，能雇到更好的程序员，因为只会吸引肯花功夫去学的人。对程序员而言，悖论更明显：若想找好工作，该学的是人们**不为找工作**而学的语言。

迄今只有少数公司想通；这里也有选择效应：它们正是程序员最想去的公司。谷歌招 Java 岗位时，也要求会 Python。

我认识一位几乎通晓所有常用语言的朋友，项目多用 Python。他说主要原因是喜欢源码**长什么样**。这听起来轻浮，其实不然：编程时**读代码的时间多于写代码**。你像雕塑家推泥一样推源码块；让源码丑陋的语言，对苛刻的程序员就像满是疙瘩的泥对雕塑家一样抓狂。

一提丑陋源码，人自然想到 Perl。但 Perl 表面的丑不是我说的那种。**真正的丑**不是语法看着凶，而是用错误概念搭程序。Perl 像卡通人物骂街，但有些场合在概念上胜过 Python——至少到目前为止。

两种语言当然都在变。但它们与 Ruby（以及 Icon、Joy、J、Lisp、Smalltalk）共有的一点，是**由真正在乎编程的人创造并使用**——而这些人往往也做得好。
