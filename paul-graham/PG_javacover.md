# Java 的封面

> 原文：<https://paulgraham.com/javacover.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：Java's Cover
> 年代：2001 年 4 月

这篇文章源于我和几位程序员关于「Java 为何闻起来可疑」的交谈。**这不是对 Java 的批评！** 它是一则关于**黑客雷达**的案例研究。久而久之，黑客会对好技术、坏技术养成嗅觉。我想试着写下，是什么让 Java 在我眼里显得可疑。

读过的人里，有人觉得这是尝试书写一种此前少有人写的话题；也有人说，我会因为看似在写自己不懂的东西而惹麻烦。所以，以防万一，我先澄清：**我在这里写的不是 Java**（我从未用过），而是**黑客雷达**（我想了很多）。

「不能以封面辨书」这句格言，诞生于书籍以素纸板封面出售、由买家自行装订的年代。那时，确实不能以封面辨书。但出版业已进步：今天的出版商努力把封面做成你能「读」出内容的东西。

我在书店泡了很多时间，自觉已能读懂出版商想告诉我的一切——也许还多懂一点。没在书店度过的时光，多半花在电脑前；某种程度上，我也学会了**凭「封面」判断技术**。也许是运气，但我因此躲开了几种后来证明真是烂泥的技术。

到目前为止，Java 在我闻起来像烂泥。我从没写过 Java 程序，参考书也只是略翻，但我有预感它不会是非常成功的语言。我也许会错——对技术做预测是危险生意。但权当作时间胶囊，记录我为何不喜欢 Java 的「封面」：

### 1. 它被过度炒作

真正的标准不必推销。没人需要推销 C、Unix 或 HTML。真正的标准，在大多数人听说它之前，往往已经确立。在黑客雷达上，Perl 凭自身实力就和 Java 一样大，甚至更大。

### 2. 定位偏低

在最初的 Java 白皮书里，Gosling 明确说，Java 的设计不会让习惯 C 的程序员觉得太难——它被设计成另一种 C++：C 加上从更高级语言借来的几个想法。像情景喜剧、垃圾食品、包价旅游的设计者一样，Java 的设计者是有意识地为**不如他们聪明**的人设计产品。历史上，为别人设计的语言往往很糟：Cobol、PL/I、Pascal、Ada、C++。好语言是为创造者自己设计的：C、Perl、Smalltalk、Lisp。

### 3. 另有图谋

有人说过，若人们只因有话要说而写书，而非因为想写书，世界会更好。同样，我们不断听到 Java，不是因为编程语言本身有什么要说——我们听到 Java，是 Sun 削弱微软计划的一部分。

### 4. 没人爱它

C、Perl、Python、Smalltalk、Lisp 程序员都爱自己的语言。我从没听人说过爱 Java。

### 5. 人们被迫使用

我认识的使用 Java 的很多人，是因为觉得**不得不**用：要么以为融资需要，要么以为客户想要，要么管理层要求。他们都是聪明人；若技术真的好，他们会自愿使用。

### 6. 厨子太多

最好的编程语言由小团队开发。Java 似乎由委员会运作。若它最终成为好语言，将是历史上委员会第一次设计出好语言。

### 7. 官僚气重

就我对 Java 的有限了解，做事似乎有很多协议。真正好的语言不是这样——它们让你做想做的事，然后让开。

### 8. 伪草根

Sun 现在假装 Java 是像 Perl 或 Python 一样的草根开源语言项目——只是恰好由一家巨头控制。于是这门语言很可能和大公司出品的其他东西一样，单调而笨拙。

### 9. 为大型组织设计

大型组织的目标与黑客不同。它们想要（被认为）适合大型平庸程序员团队的语言——带有像 U-Haul 卡车限速器一样的特性，防止蠢人造成太大破坏。黑客不喜欢居高临下对他们说话的语言；黑客只要**力量**。历史上，为大型组织设计的语言（PL/I、Ada）输了，黑客语言（C、Perl）赢了。原因：今天的少年黑客是明天的 CTO。

### 10. 不对的人喜欢它

总体上，我最敬佩的程序员并不被 Java 吸引。谁喜欢 Java？穿西装的人——分不清一种语言和另一种，但知道媒体上总在提 Java；大公司的程序员——惊讶地发现还有比 C++ 更好的东西；以及死记硬背的本科生——随时准备喜欢任何可能帮他们找到工作的东西（这会在考试里考吗？）。这些人的意见随风而变。

### 11. 它的「爹」处境艰难

Sun 的商业模式正从两面被侵蚀：与台式机同类的廉价 Intel 处理器，对服务器已足够快；FreeBSD 作为服务器 OS 似乎至少不逊于 Solaris。Sun 的广告暗示工业级应用需要 Sun 服务器——若果真如此，Yahoo 应第一个买 Sun；但我在那里工作时，服务器全是跑 FreeBSD 的 Intel 机器。这对 Sun 的前景不祥；若 Sun 遇困，可能拖 Java 下水。

### 12. 国防部喜欢它

国防部鼓励开发者使用 Java。在我看来，这是最坏的信号。国防部保卫国家做得不错（尽管昂贵），但他们爱计划、程序和协议；他们的文化与黑客文化相反——在软件问题上，他们往往会押错注。国防部上一次真正喜欢的编程语言，是 Ada。

请记住，**这不是对 Java 的批评，而是对其封面的批评。** 我对 Java 了解不够，谈不上喜欢或不喜欢。这只是解释我为何没有热切去学它。

在甚至没试着写过程序之前就放弃一门语言，似乎草率。但这是所有程序员都必须做的事——技术太多，学不过来。你得学会凭外在迹象判断哪些值得花时间。我同样草率地放弃过 Cobol、Ada、Visual Basic、IBM AS/400、VRML、ISO 9000、SET 协议、VMS、Novell Netware、CORBA 等等——它们只是闻起来不对。

Java 的情况，我也许错了。也许一门由大公司推广以制衡竞争对手、由委员会为「主流」受众设计、被炒作上天、又为国防部所爱的语言，碰巧仍是干净、优美、强大、我会爱用的语言——有可能，但似乎极不太可能。
