# 会反击的过滤器

> 原文：<https://paulgraham.com/ffb.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：Filters that Fight Back
> 年代：2003 年 8 月

我们也许可以通过让贝叶斯垃圾邮件过滤器**跟踪邮件中的链接**——看看另一端到底是什么——来提高其准确性。death2spam 的 Richard Jowsey 现在已经在边界案例中这样做了，据报告效果很好。

为什么只在边界案例中这样做？而且为什么只做一次？

正如我在《过滤器会杀死垃圾邮件吗？》中提到的，跟踪垃圾邮件中的所有 URL 会产生一个**有趣的副作用**。如果主流邮件客户端为了过滤垃圾邮件而这样做，垃圾邮件发送者的服务器将承受**沉重的打击**。我越想越觉得这是个好主意。这不仅有趣——**很难想出比这更精准的对垃圾邮件发送者的反击了。**

因此我想建议那些开发垃圾邮件过滤器的人：增加一个**"惩罚"模式**——如果启用，它会对疑似垃圾邮件中的每一个 URL 自动爬取 n 次，n 由用户设定。[1]

很多人指出过，当前邮件系统的一个问题是**它太被动了**——你让它做什么它就做什么。到目前为止，所有修复方案似乎都需要新的协议。而这个方案不需要。

如果被广泛采用，自动爬取的垃圾邮件过滤器将使邮件系统**产生"回弹"效应**。垃圾邮件的巨大发送量，迄今一直对发送者有利，但现在将反过来打击他——就像一根树枝弹回打在他脸上。自动爬取过滤器会**推高发送者的成本、降低他的销售**：他的带宽使用量会暴增，服务器会在负载下瘫痪——这将使它们对那些本来会响应垃圾邮件的人也变得不可用。


### 发出一百万封邮件每小时，就承受一百万次访问每小时。

我们需要确保这只针对疑似垃圾邮件。通常来说，任何被发送给数百万人的 URL 很可能就是垃圾 URL，所以对每封邮件中的每个 HTTP 请求都这样做几乎总是没问题的。但有少数例外：比如 Yahoo Mail 和 Hotmail 等免费邮件服务发出的邮件底部的 URL。

为了保护这类网站并防止滥用，自动爬取应与**垃圾广告网站黑名单**结合使用。只有黑名单上的网站才会被爬取，而且网站只有经人工审核后才会被列入黑名单。一封垃圾邮件的生命周期至少有几个小时，因此应该来得及更新列表来干扰推广新网站的垃圾邮件。[2]

大量自动爬取只对高带宽连接的用户可行，但这样的用户已经足够给垃圾邮件发送者制造严重麻烦。事实上，**这个方案完美地镜像了问题本身。**垃圾邮件的问题在于，为了触达少数容易上当的人，发送者给所有人发了邮件。不上当的收件人只是附带损害。但不上当的多数人要停止收到垃圾邮件，就必须能阻止（或威胁阻止）上当者的响应行为。自动爬取的垃圾邮件过滤器为他们提供了这样做的方式。

这能杀死垃圾邮件吗？不完全能。最大的垃圾邮件发送者大概能保护他们的服务器。但对他们来说最简单、最便宜的做法是**在邮件中包含有效的退订链接**。对小规模发送者和那些雇佣垃圾邮件发送者做推广的"合法"网站来说更是如此。因此，如果自动爬取过滤器普及开来，它们实际上会变成**"自动退订"过滤器**。

在这个场景下，垃圾邮件将像操作系统崩溃、病毒和弹窗广告一样，成为一种**只困扰那些不愿意使用正确软件的人的瘟疫**。

**注释**

[1] 自动爬取过滤器需要跟踪重定向，在某些情况下（比如页面只写着"点击这里"）还需要跟踪多层链接。同时要确保 HTTP 请求与主流浏览器的请求无法区分，包括请求头的顺序和 Referer。如果响应在 x 时间内没有返回，默认给出一个较高的垃圾邮件概率。与其让 n 为常数，不如让它成为已见过的提及该网站的垃圾邮件数量的函数。这将增加一层防止滥用和意外的保护。

[2] 本文最初使用"白名单"而非"黑名单"一词。虽然它们的运作方式像黑名单，但我倾向于称之为白名单，因为这也许能减少法律攻击的风险。但这似乎只是让读者困惑了。黑名单可能应该有多个。单一故障点既容易被攻击也容易被滥用。
