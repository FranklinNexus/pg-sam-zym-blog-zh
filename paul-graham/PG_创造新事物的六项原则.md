# 创造新事物的六项原则

> 原文：<https://paulgraham.com/newthings.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：Six Principles for Making New Things
> 年代：2008 年 2 月

[Arc](https://paulgraham.com/arc.html) 发布时招来的激烈反应，带来一个我没料到的后果：让我意识到，自己其实有一套 **设计哲学**。批评里最清楚的一类，是说 Arc 太「轻薄」——折腾好几年，就几千行宏？怎么不去碰更实在的大问题？

我琢磨这些评论时忽然觉得耳熟：几乎就是当年人们对 Viaweb、Y Combinator，以及我大多数文章的第一反应。

### 同样的嘲笑，同样的结局

我们推出 Viaweb 时，在风投和「电商专家」眼里像个笑话：1995 年窝在公寓里的两个年轻人并不酷；做出来的东西在他们看来甚至不算软件——软件等于庞大的 Windows 应用。Viaweb 是他们见过的第一个 Web 应用，看起来像个普通网站；听说我们第一年还不处理信用卡（确实如此），更是嗤之以鼻。在他们心里，电商的「核心」是支付系统——听起来又复杂又严肃。

然而 Viaweb 不知怎么就干掉了所有对手。

Y Combinator 上线时的反应几乎一模一样：听起来轻得可笑。那时的创业融资意味着 A 轮——几百万美元、数月严肃会议、电话本厚的协议，给一小撮履历光鲜的创业者。相比之下 YC 显得微不足道。现在还不好说 YC 会不会像 Viaweb 那样收场，但从模仿者的数量看，很多人似乎相信我们摸到了什么。

我的文章成功与否，除了页面浏览量很难衡量；但至少反应和起初不同了。早年 Slashdot 上的喷子（译成通顺话）大概是：这人是谁？凭什么写这些？我没读文章，但这么短、这么随便的东西，不可能比学者们写的厚书更有料。现在换了一代网站、一代喷子，至少他们省略了开头的「这人是谁」。

如今人们对 Arc 说的话，和当初对 Viaweb、YC、我的文章一样。为什么总是这个模式？因为我在这四件事上的 **做法其实相同**。

### 六项原则

我喜欢：

1. **找简单的解法**
2. **对准被忽视的问题**
3. **且那问题确实值得解决**
4. **用尽可能不正式的方式交付**
5. **从非常粗糙的 1.0 起步**
6. **然后快速迭代**

我第一次把这六条写清楚时，注意到一件扎眼的事：这几乎是 **招来轻蔑的配方**。简单解法虽然更好，却不如复杂解法显得厉害；被忽视的问题，按定义就是多数人觉得不重要；不正式交付意味着别人不能靠包装判断，必须真的读懂——更费脑子；粗糙的 1.0 则永远像未完成的小作品。

我当然早就注意到，人起初总抓不住新想法，一度以为只是因为多数人不够聪明。现在明白还有更深机制：像逆向投资策略一样，照这套做，你几乎总在旁人眼里 **做错了**。

### 为什么「错」反而有效

正如逆向投资，要点正在于此：你拿到了别人为显得「正统」而放弃的全部优势。

- 做被忽视的问题 → 更可能发现新东西，因为竞争少。
- 不正式地交付 → 省下把它包装得吓人的精力；也减少 **连自己都骗过** 的风险。
- 先出粗糙 1.0 再迭代 → 解法还能借 **自然的想象力**——费曼说过，那比你自己强得多。

以 Viaweb 为例：简单解法是软件跑在服务器上；被忽视的问题是 **自动生成网站**——1995 年网店都靠设计师手工做，我们知道这无法扩展；真正要紧的是界面设计，不是支付；不正式交付是我穿牛仔裤 T 恤去商家办公室；粗糙 1.0 上线时，若没记错，代码不到一万行。

这套做法的力量不限于创业、语言或文章，几乎适用于一切创造性工作——绘画亦然：塞尚和克利就是这么干的。

### YC、Reddit 与「显而易见」

在 Y Combinator，我们用资金为这套方法下注，也鼓励被投公司这样工作。新点子总在鼻子底下。去找 **简单却被别人忽略** 的东西——后人会说「这不是显而易见吗」的那种，尤其当他们被过时惯例或表面华丽带偏时。弄清 **真问题** 并解决它。别忙着装「像家公司」；长期赢的是产品。**越早发布越好**，才能从用户身上学到你该做什么。

**Reddit** 是典范。刚上线时很多人觉得「啥也没有」；对不懂设计的人来说，刻意极简像根本没设计。但 Reddit 解决了真问题：告诉你什么是新的，其余别挡路。于是爆发式增长。等主流观念追上，一切显得理所当然。人们看 Reddit 以为创始人运气好——像所有这类事一样，其实比看起来难得多：他们逆流推得太狠，把水流方向都改了；现在你才觉得他们在顺流而下。

所以当你看着 Reddit 想「真希望我也能想出这种点子」——记住：**这样的点子到处都是**。你只是忽略它们，因为它们 **看起来不对**。
