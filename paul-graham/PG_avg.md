# 击败平均水平

> 原文：<https://paulgraham.com/avg.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：Beating the Averages
> 年代：2001 年 4 月（修订 2003 年 4 月；本文摘自 2001 年 Franz 开发者研讨会演讲。）

### Viaweb 与 Lisp

1995 年夏天，我和 Robert Morris 创办 Viaweb，做让终端用户自建网店的软件。当时的新意在于：跑在我们服务器上，用普通网页当界面。很多人可能同时想到这点，但据我所知 Viaweb 是第一个 Web 应用——我们因此命名：经 Web，而非跑在桌面。

另一不寻常之处：主要用 Lisp 写成。它是首批面向最终用户的大型 Lisp 应用之一；此前 Lisp 多用于高校与实验室。[1]

### 秘密武器

Eric Raymond 在《如何成为黑客》里建议学什么语言：从 Python、Java 入手；认真者再学 C（搞 Unix）和 Perl（系统管理、CGI）；真正严肃的还应考虑 Lisp——值得学，因为一旦「悟了」会有深刻启蒙，此后一生成为更好的程序员，即使用 Lisp 不多。

这像学拉丁语的论据：除当古典学教授外难找工作，但能锻炼思维、让你用英语等语言写得更好。但比喻不能拉太远：拉丁语没人说，写了没人懂；Lisp 是计算机语言，机器会说程序员让它们说的任何语言。

若 Lisp 真让你成为更好的程序员，为何不用？若画家有一支能让他画得更好的笔，难道不该每幅画都用？我不是取笑 Raymond——他的建议大体好，他对 Lisp 的评价几乎是传统看法。但传统看法有矛盾：**Lisp 让你成为更好的程序员，你却不会用它。**

### 语言只是工具？

编程语言只是工具。若 Lisp 真能产出更好的程序，就该用；若不能，谁需要它？这不只是理论问题。软件业竞争极强、易现自然垄断；写得更快更好的公司会把对手挤出市场。初创对此尤其敏感：全有或全无，押错技术会被碾碎。

Robert 和我都很熟 Lisp，看不出有理由不信直觉、选 Lisp。我们知道别人用 C++ 或 Perl——但那没有意义；若按「别人用什么」选技术，你会跑 Windows。**选技术要忽略别人在做什么，只考虑什么最有效。** 初创尤其如此：大公司可以学别的公司；初创不能学别的初创——很多人连这点都没意识到。

大公司平均年增约 10%；你若完全按大公司方式做事，绩效也就平均——年增 10%。初创也一样：按普通初创方式做事，绩效就平均。**而平均绩效意味着倒闭**——初创存活率远低于 50%。所以你在做初创，最好做些古怪的事；否则有麻烦。

### Web 应用解放了语言

1995 年我们就知道一件事——我认为竞争对手不懂，现在仍少有人懂：**软件只需跑在自己服务器上时，可以用任何语言。** 写桌面软件则强烈倾向与操作系统同语；十年前写应用意味着写 C。基于 Web 的软件，尤其当你同时拥有语言与 OS 源码时，可以用任何语言。

新自由是双刃剑：现在必须想选哪一种。假装什么都没变的公司，会发现竞争对手没有假装。若随便选，你选什么？**我们选 Lisp。**

### 为何选 Lisp

显然，这个市场里开发速度极重要。大家都从零开始，谁先做完功能谁占优。我们知道 Lisp 极适合快速写软件；服务器应用放大快速开发——做完即可发布。别人不用 Lisp 更好：可能带来技术优势，而我们急需一切帮助。

创办 Viaweb 时我们没有商业经验：不懂营销、招聘、融资、获客；两人都没做过所谓「正经工作」。我们只会写软件，希望这能救命；软件部门有任何优势都要抓。

用 Lisp 可说是一次实验：假设用 Lisp 能比对手更快完成功能，还能做他们做不到的事；Lisp 够高层，团队可小、成本更低——于是更好产品、更低价格、仍盈利，最终拿走所有用户、对手归零。我们希望如此。

### 实验结果

结果有点意外：**成了。** 最终约二三十家竞争对手，没有一家的软件打得过我们。我们有跑在服务器上、却像桌面应用的所见即所得网店构建器；对手是 CGI 脚本。功能上我们始终领先。

有时对手绝望地推我们还没有的功能；Lisp 下开发周期快到，对手发新闻稿后一两天内我们常能复制。记者打电话来时，我们往往也有了。

对手一定以为我们有秘密武器——在破译他们的 Enigma。确实有，但比他们想得更简单：没人向我们泄密，只是我们开发软件的速度超出所有人想象。

### 《豺狼的日子》

我九岁时读过弗雷德里克·福赛思的《豺狼的日子》：刺客要杀法国总统，得穿过警察上到能俯瞰路线的公寓。他扮拄拐老人走过去，警察毫无怀疑。

我们的秘密武器类似：用一种怪异的 AI 语言写软件，语法满是括号。多年听人这样描述 Lisp 让我很烦，此刻却成优势。**商业里，竞争对手不懂的技术优势无价；商业与战争一样，意外与武力同样值钱。**

因此——有点不好意思说——Viaweb 期间我从未公开谈论 Lisp。没对媒体提，在站上搜 Lisp 只会看到我简介里的两本书名。这不是偶然：初创应尽量少给竞争对手信息。若他们不知道或不在乎我们用什么语言，我想保持如此。[2]

最懂我们技术的是客户。他们也不在乎语言，但注意到软件极好——几分钟就能建漂亮网店。口碑带来越来越多用户：1996 年底约 70 家店，1997 年底 500，六个月后雅虎收购时 1070。今日 Yahoo Store 仍主导该市场，是雅虎较赚钱的一块，其上店铺是 Yahoo Shopping 的基础。我 1999 年离开雅虎，不知现在用户数，最后听说约两万。

### Blub 悖论

Lisp 好在哪里？若这么好为何人人不用？听起来像修辞，其实有直白答案：Lisp 好不是因为信徒才看得见的魔力，而是**现有语言里它最强大**；人人不用是因为语言不仅是技术，也是思维习惯，而习惯变得极慢。两点都需展开。

先说一句惊世之言：**编程语言威力不同。** 很少有人否认高级语言比机器语言强；你不该手写机器码，而该用高级语言再编译——这观念已写入硬件：八十年代起指令集为编译器而非人手设计。大家都知道不该整程序手写汇编；较少被概括的是：**若可选多种语言，其他条件相同时，不用最强的是错——与手写机器语言同类，只是程度较轻。** [3]

例外很多：须与某语言写的程序紧密协作，可能应用同语；只做很简单的事（数值、位操作），可用较抽象度低的语言，可能稍快；一次性小程序，用库最好的语言即可。但对应用软件，应用你能得到的**最强（且合理高效）**的语言；否则是错。

机器语言显然很低级。但社会惯例常把高级语言当等价——它们不是。「高级语言」技术上并无清晰分界；语言沿抽象度连续体排列 [4]，从最强到各层机器语言。

Cobol 也是高级语言（会编译成机器码）。谁认真说 Cobol 与 Python 等价？它更接近机器语言。Perl 4 到 5 加了词法闭包，多数 Perl 黑客同意 5 更强——一旦承认，就承认一种高级语言可以比另一种更强；除特殊情况外，应用最强的。

这结论很少贯彻到底：到一定年纪，程序员很少自愿换语言；习惯什么就觉得什么「刚好够好」。为不伤感情，我用假想语言 **Blub** 说明——在抽象度连续体中间，比 Cobol、机器语言强，但不是最强。

Blub 程序员不会用机器语言（有编译器），也不会用 Cobol——「连 x 都没有」（x 为 Blub 的某种特性）。他往下看，知道更弱的语言缺特性；往上看，却**看不出是在往上**——只见怪异语言，觉得与 Blub 差不多，只是多了些累赘。Blub 对他够好，因为他用 Blub 思考。

站在更高语言往下看，则觉得 Blub 连 y 都没有，什么都做不了。归纳：**只有理解最强语言的人，才能看见各语言威力差别**（这或许就是 Raymond 说 Lisp 让你更好的意思）。别人的意见不可信——**Blub 悖论**：他们满足于当前语言，因为语言支配他们如何想程序。

我高中用 Basic 写程序，Basic 甚至没有递归；很难想象不用递归，当时却不觉得缺——我用 Basic 思考，且自以为精通。

Raymond 推荐的五种语言落在威力连续体不同位置，相对位置是敏感话题。我认为 Lisp 在顶；支撑之一：看另外四种我常想——**没有宏你怎么干活？** [5]

许多语言有「宏」，Lisp 宏独一无二，与括号有关——设计者不是为了标新立异；对 Blub 程序员 Lisp 代码看着怪，括号是 Lisp 与其他语言根本差异的外在证据。

Lisp 代码由 Lisp 数据构成——不是 trivial 的「源文件是字符、字符串是类型之一」；经解析器读入后，是你可以遍历的数据结构。若懂编译器，实质不是 Lisp 语法怪，而是 **Lisp 几乎没有语法**：你在其他语言解析时生成的 parse tree 里写程序，而这些树对你的程序完全可访问、可操纵——在 Lisp 里叫**宏**，即写程序的程序。

写程序的程序？用 Cobol 想，不常需要；用 Lisp 想，一直需要。此处无法举一个强宏让你秒懂——对不懂 Lisp 的人只是天书；《ANSI Common Lisp》我尽量快进，宏仍到 160 页才讲。

但可以论证：Viaweb 编辑器源码约 20–25% 是宏。宏比普通 Lisp 函数难写，不必要用宏被视为坏品味——故每个宏都确有必要：至少五分之一代码在做其他语言很难做的事。再怀疑 Lisp 神秘力量的 Blub 程序员也该好奇——我们不是自娱，是极小初创，拼命编程，要在技术上隔开对手。

可疑的人会想：是否有关联？大块代码在做别语言极难之事，成品做了对手做不到的事。也许有联系。我鼓励你顺着这条线想——拄拐老人或许不止表面那样。

### 初创的合气道

我不指望说服（25 岁以上）的人去学 Lisp。本文目的不是改主意，而是安抚**已经想用 Lisp、却担心不流行**的人：竞争情境下，那是优势——**威力被竞争对手不懂所放大。** 若你在初创用 Lisp，别愁没人懂，应希望继续如此——很可能如此。语言本性让人满足于手头之物；硬件变太快、个人习惯太慢，实践常落后处理器十年二十年。MIT 六十年代初就在用高级语言，许多公司八十年代还在写机器码——直到处理器像急着打烊的酒保，换 RISC 把他们撵出去。

通常技术变很快；编程语言不同——不仅是技术，也是程序员所想，**半技术半宗教**。[6] 中位语言（中位程序员用的那种）像冰山一样慢。垃圾收集（Lisp 约 1960）现已被认为好；运行时类型检查也在流行；词法闭包（Lisp 七十年代初）刚上雷达；宏（Lisp 六十年代中）仍是未知大陆。

中位语言惯性巨大。我不是让你对抗这股力，而是相反：像合气道，**借力打力**。在大公司难说服尖头老板让你用 Lisp——他刚在报上看到某语言要像 Ada 二十年前那样统治世界。但若你在还没有尖头老板的初创，可以像我们一样利用 Blub 悖论：用粘在「中位语言」上的对手永远追不上的技术。

若你在初创，评估竞争对手有个窍门：**看招聘启事。** 网站其他内容可能是图库照片或套话，招聘必须具体，否则招错人。Viaweb 年间我读大量招聘：每月似乎冒出新对手。我先看有没有在线 demo，再看招聘。几年后我能分辨该担心谁：招聘 IT 味越重，公司越不可怕。要 Oracle 经验的最安全；要 C++ 或 Java 也安全；要 Perl 或 Python 有点吓人——技术侧像真黑客在管；若见过招 Lisp 黑客，我会非常担心。

## 注释

[1] Viaweb 起初两部分：Lisp 写的编辑器（建站）与 C 写的订购系统。第一版以 Lisp 为主，订购系统很小。后来加了 C 的图像生成器、主要 Perl 的后台管理。2003 年 1 月雅虎发布 C++/Perl 新版编辑器；很难说是否「不再用 Lisp」——译成 C++ 实则要写 Lisp 解释器；据我所知页面模板源文件仍是 Lisp（参见 Greenspun 第十规则）。

[2] Robert Morris 说我不必保密：就算对手知道我们用 Lisp，也不会懂为何——「他们若那么聪明，早该用 Lisp 了。」

[3] 图灵等价意义上所有语言同样强，但这不是程序员关心的「强」。程序员关心的强，可理解为：弱语言只能通过为强语言写解释器才能获得的能力。A 有去空格运算符、B 没有，未必使 A 更强（B 可写子程序）；若 A 支持递归、B 不支持，则难用库补上。

[4] 对书呆子：或像朝顶部变窄的格；重要的是存在偏序，而非形状。

[5] 把宏当独立特性有点误导；实践中词法闭包、剩余参数等大大增强宏的用处。

[6] 因此语言比较要么像宗教战争，要么像绝对中立的本科教材——实则人类学。珍惜和平或想要终身教职的人会回避；但问题只有一半是宗教，若你想设计新语言，确有可研究之物。
