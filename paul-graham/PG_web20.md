# Web 2.0

> 原文：<https://paulgraham.com/web20.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：Web 2.0
> 年代：2005 年 11 月

「Web 2.0」有意义吗？直到最近我还以为没有，但真相更复杂些。起初，是的，**毫无意义**。现在它似乎有了含义。而那些讨厌这个词的人，大概也是对的——因为如果它的意思真如我所想，我们根本不需要它。

我第一次听到「Web 2.0」，是在 2004 年那场同名会议的标题里。当时它本意是「**把网络当作平台**」，我理解成基于网络的应用。[1] 所以今年夏天，Tim O'Reilly 主持一场专门给「Web 2.0」下定义的会议时，我很意外：这不已经等于「把网络当平台」了吗？要是本来就没意思，我们何必还要这个短语？

### 起源

Tim 说，「Web 2.0」一词最早出现在「O'Reilly 与 Medialive International 的一次头脑风暴」里。Medialive International 是什么？官网写：「技术贸易展与会议的制作方。」所以那场头脑风暴，八成就是在商量：O'Reilly 想办一场关于网络的会，该起什么名。

我不认为他们有意暗示网络有了**新版本**。他们只想强调：网络又重要了。这是一种语义上的「赤字开支」：他们知道新东西要来，「2.0」指的就是那些东西最终会是什么。他们猜对了，新东西确实来了。但版本号在短期内带来了尴尬：为第一次会议写宣传稿时，总得有人试着解释「2.0」指什么。不管具体指什么，「网络作为平台」至少不算太窄。

「Web 2.0 = 网络作为平台」这套说法，第一次会议后就没撑多久。到第二次会议，「Web 2.0」似乎变成了**民主**——至少人们在网上这么写的时候是这样。会议本身并不草根：门票 2800 美元，能去的主要是风投和大公司的人。可奇怪的是，Ryan Singel 在 Wired News 上写这场会，居然提到「**成群极客**」。我朋友去问 Ryan，他自己也懵了：他原稿大概是「成群的风投和商务拓展人士」，后来缩成「成群的人」，编辑再扩成「成群的极客」。毕竟 Web 2.0 会议嘛，想必满是极客，对吧？

嗯，不。大概七个。连 Tim O'Reilly 都穿了西装——那画面陌生到我一时无法解析。我看见他走过，对 O'Reilly 的人说：「那家伙长得真像 Tim。」「哦，那就是 Tim，他买了套西装。」我追上去，果然是他；他说刚在泰国买的。

2005 年的 Web 2.0 会议让我想起泡沫时期的互联网展会：风投四处逡巡，找下一个热门初创公司。一大群人**决不错过**什么，营造出同样古怪的气氛。错过什么？他们不知道。无论将要发生什么——无论 Web 2.0 最后是什么。

我不会只因风投又热衷投资，就把它叫作「泡沫 2.0」。互联网**确实是**大事。萧条与繁荣一样，都是过度反应。走出萧条之后，这个领域大幅增长是意料之中——就像大萧条前跌得最狠的行业，反弹也最猛。

它不会变成第二次泡沫，是因为 **IPO 市场没了**。风投被退出策略驱动。90 年代末他们资助那些可笑的初创公司，是因为指望卖给容易上当的散户投资者，一路笑到银行。这条路堵死了。现在默认退出是被收购，而收购方比 IPO 投资者更不容易非理性亢奋。最接近泡沫估值的，大概是鲁珀特·默多克花 5.8 亿美元买 Myspace——也就差个十倍左右。

### Ajax

「Web 2.0」除了会议名字，还有别的意思吗？我不愿承认，但确实开始有了。现在有人说「Web 2.0」，我大概知道指什么。我**既鄙视**这个短语又听得懂——这恰恰证明它开始有意义了。

含义之一肯定是 **Ajax**——我至今仍只能勉强接受不加引号地写它。基本上，「Ajax」的意思就是：**Javascript 现在能用了**。于是基于网络的应用，可以做得更像桌面应用。你读这篇文章时，正有一整代新软件在利用 Ajax 编写。自微型计算机出现以来，还没见过这样的应用浪潮。连微软都看到了，但对他们来说已经太晚，顶多泄露些「内部」文件，营造「我们跟上了」的印象。

事实上，新一代软件写得**太快**，微软连引导都来不及，更别说自己做。他们现在唯一的希望，是在 Google 之前买下所有最好的 Ajax 初创公司——这也不容易，因为 Google 在收购微型初创公司上的先发优势，就像几年前在搜索上一样。毕竟，Google 地图——典型的 Ajax 应用——就是收购一家初创公司的结果。

讽刺的是，Web 2.0 会议的原始描述**部分说对了**：基于网络的应用是 Web 2.0 的重要组成。但我相信他们是**误打误撞**。Ajax 热潮直到 2005 年初 Google 地图出现、并**造出**「Ajax」一词才开始。

### 民主

Web 2.0 的第二大要素是**民主**。我们已有若干例子证明：业余者在有合适系统引导时，可以超越专业人士。维基百科或许最出名。专家给它的评价平平，但他们漏了关键点：**它已经够好了**。而且免费——意味着人们真的会读。在网上，要付费才能读的文章，等于不存在；就算你愿意自己掏钱，也没法链接过去，它们进不了对话。

民主似乎还在另一处获胜：**什么算新闻**。除了 Reddit，我几乎不看任何新闻站。[2] 我知道若有大事，或有人写了特别有意思的文章，它会出现在那里。何必再去翻某家报纸或杂志的首页？Reddit 像整个网络的 RSS，外加质量过滤器。类似站点还有 Digg（技术新闻站，人气迅速逼近 Slashdot）和 del.icio.us（协作书签网络，点燃了「标签」运动）。维基百科的吸引力主要是够好且免费；这些站点则表明，**投票者比人类编辑好得多**。

Web 2.0 民主最戏剧性的例子，不在**选**思想，而在**产**思想。我早注意到：个人网站上的东西，往往不亚于、甚至优于报纸杂志。现在我有了独立证据：Reddit 热榜链接，多半是个人站点，而非杂志文章或新闻报道。

我为杂志写稿的经历提供了一个解释：**编辑**。他们决定你能写什么，而且通常能改你写的一切。结果是**极端被压平**。编辑带来第 95 百分位的写作——95% 的文章因编辑变好，5% 却被拖累。那 5% 里，你就会遇到「成群极客」。

在网上，人们想发什么就发什么。几乎一切都达不到印刷品里经编辑润色后的水准。但作者池**极其**大。只要池子够大，缺少阻尼就意味着：**网上最好的写作，应能超过印刷品里最好的**。 [3] 现在网络也进化出了选好内容的机制，**净效果是网络赢**。选择战胜阻尼，就像市场经济战胜中央计划。连初创公司这次也不一样。

它们相对泡沫时期的初创公司，就像博主相对印刷媒体。泡沫里，「初创公司」指一家由 MBA 领导、字面意义上烧掉几百万风投、要「快速做大」的公司。现在指更小、更年轻、更技术的一群人，刚决定做点很牛的东西；要不要融风投级别的钱，以后再说——若要，也是**按他们的条件**。

### 不要虐待用户

我想大家都会同意：民主和 Ajax 是「Web 2.0」的要素。我还看到第三点：**不要虐待用户**。泡沫时期，很多热门网站对用户相当专横——不只是明显的方式，比如强迫注册、塞烦人广告。90 年代末，普通网站的**设计本身**就是一种虐待：许多热门站堆满碍眼品牌，加载慢，还向用户传达：这是我们的站，不是你的。（笔记本上的 Intel、Microsoft 贴纸，有实体版类比。）

我认为根源在于：网站觉得自己在**免费送**东西；而直到最近，免费送东西的公司往往可以对用户很横。有时到了经济虐待狂的地步：站主以为给用户越痛，对自己越有利。这种模式最刺眼的残余，可能在 salon.com——你能读故事开头，要看全文得坐着看一段**电影**。

我的建议是：初创公司永远不要对用户摆架子。除非要为用户存东西，否则别让他们注册。若必须注册，别让他们等邮件里的确认链接；事实上，除非真有必要，连邮箱都别问。别问任何不必要的问题。除非用户明确要求，否则别发邮件。别把链出去的页面嵌在框架里，也别在新窗口打开。若有免费版和付费版，别让免费版限制太多。若你发现自己在问「该不该允许用户做 X？」——拿不准就答**是**。宁可慷慨。在《如何创办一家初创公司》里，我建议初创公司永远别让任何人在自己**下方飞过**——即别让别的公司提供更便宜、更简单的方案。另一种「飞低」是给用户更多权力：让用户做他们想做的；你若不做而对手做了，你就麻烦了。

从这个意义上，iTunes 有点 Web 2.0 的味道：终于可以买单曲，不必整张专辑。唱片业恨这个，能拖就拖。但用户要什么很明显，所以 Apple 从唱片公司**下方飞过**。[4] 不过说真的，把 iTunes 叫 Web 1.5 或许更准确。Web 2.0 用在音乐上，大概意味着各乐队免费赠送无 DRM 的歌曲。

对用户最好的方式，是**免费送**竞争对手收费的东西。90 年代很多人以为，我们现在该有小额支付系统了。事实上方向相反：最成功的站，是那些找到新办法免费送东西的站。Craigslist 大体上已摧毁 90 年代的分类广告站；OkCupid 看起来会对上一代约会站做同样的事。提供网页服务**非常**便宜；若每次浏览哪怕只赚一点点，就能盈利。定向广告技术也在进步。十年后 eBay 被广告支持的 freeBay（或更可能是 gBay）取代，我不会惊讶。

听起来怪，但我们告诉初创公司：**尽量少赚钱**。若你能把十亿美元的产业变成五千万美元的产业，而且五千万都归你，那就更好。事实上，把东西做便宜往往最终赚更多，就像自动化往往最终创造更多就业。终极目标是微软。有人用免费的基于网络的 MS Office 替代品戳破那个气球时，会发出多大的响声。[5] 谁来做？Google？他们似乎在慢慢来。我怀疑那根针会握在一对 20 岁的黑客手里——他们太天真，不会被这个想法吓住。（能有多难？）

### 共同主线

Ajax、民主、不鄙视用户——共同点是什么？直到最近我才意识到它们有共同点，这也是我讨厌「Web 2.0」的原因之一：它像贴在任何新事物上的标签，**不预测任何东西**。但有一条共同主线：**Web 2.0 意味着按网络本该被使用的方式去用网络**。我们现在看到的「趋势」，只是网络固有本性，从泡沫时期强加在它身上的破碎模型下浮现出来。

我读 Excite 联合创始人 Joe Kraus 的采访时意识到这一点。[6] Excite 的商业模式根本不对。我们掉进经典陷阱：新媒体出现时，先照搬旧媒体的做法、内容、商业模式——失败之后，才摸索出更合适的模式。泡沫破裂后的几年，表面像没发生什么大事。但回头看，有事在发生：**网络在找它自然的静息角**。例如民主那一块——不是某人「创新」出来的，而是网络**自然倾向于**产生的结果。

把类似桌面的应用通过网络交付，这想法几乎和网络一样老。但第一次是 Sun 接盘，我们得到 Java 小程序。此后 Java 被改造成 C++ 的通用替代品；1996 年关于 Java 的故事是：它代表新软件模型——不靠桌面应用，而跑从服务器交付的 Java「小程序」。这计划在自身重量下崩塌。微软帮了一把，但本来也会死：黑客们毫无兴趣。当你发现公关公司在把某物宣传成「下一个开发平台」，你可以确定**它不是**。若是，你不需要公关公司告诉你——黑客早就在上面写东西了，就像 Busmonster 这类站在 Google 还没打算把它当平台之前，就把 Google 地图当平台用。

Ajax 是下一个热平台的证据，是成千上万黑客**自发**在它上面构建东西。**孩子们喜欢它。**

Web 2.0 三个组成部分还有另一点共通。给个线索。假设你向投资者推销这样一个 Web 2.0 初创想法：

del.icio.us、flickr 等站让用户用描述性标记给内容「打标签」。但还有大量**隐式**标签被忽略：网页链接里的文字。而且这些链接构成社交网络，连接创建页面的个人与组织；用图论可从网络估算每个成员的声誉。我们计划挖掘这些隐式标签，连同它们体现的声誉层级，用来增强网络搜索。

你觉得投资者平均要多久才会意识到，这其实是在描述 **Google**？

Google 在 Web 2.0 三个组成部分上都是先驱：核心业务用 Web 2.0 术语描述起来酷得吓人；「不虐待用户」是「不作恶」的子集；当然 Google 地图掀起了整个 Ajax 热潮。Web 2.0 意味着按网络本该被使用的方式去用，Google 正是如此——**这是他们的秘密**。他们**顺风**而行，而不是像印刷媒体那样无风停滞、祈祷商业模式，或像微软和唱片公司那样顶风航行、起诉客户。[7] Google 不强行按自己的方式推动事情；他们设法弄清**将会**发生什么，然后安排自己届时站在那里。这是对待技术的正确方式——随着商业里技术成分越来越大，这也是做生意的正确方式。

Google 是「Web 2.0」公司，说明这个词虽有含义，也相当**空洞**。它像「对抗疗法」——只不过表示**把事情做对**；当你需要专门一个词来形容「做对」，可不是好兆头。
