# 伟大的黑客

> 原文：<https://paulgraham.com/gh.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：Great Hackers
> 年代：2004 年 7 月（本文摘自 Oscon 2004 演讲。）

### 新书与争议

几个月前我写完一本新书，书评里不断出现「挑衅」「争议」这类词，更别提「白痴」了。我并非故意写一本争议之作；我只是想提高效率，不想浪费读者时间，重复他们早已知道的事——直接给出差异（diff）就好。但这样大概注定会出一本令人不安的书。

### 财富差异即生产力差异

争议最大的观点大概是：财富差距未必像我们以为的那么成问题。书里我没有说财富差异本身是好事；我说在某些情况下，它可能是好事的**迹象**。剧烈头痛本身不好，却可能是好事的迹象——比如你被打晕后正在恢复意识。财富差异可以是生产力差异的迹象（在只有一个人的社会里，两者等同）。这几乎肯定是好事：若你的社会没有生产力差异，大概不是因为人人都是托马斯·爱迪生，而是因为你根本没有托马斯·爱迪生。

### 技术放大差距

在低技术社会里，你看不到太大的生产力差异。若有一支游牧部落捡柴生火，最好的捡柴人能比最差的强多少？两倍？而当你交给人们计算机这样的复杂工具，他们能拿它做的事，差异就巨大了。Fred Brooks 1974 年写过，他引用的研究发表于 1968 年——这并非新想法。但我认为 Brooks **低估了程序员之间的差异**：他按代码行数衡量生产力，说最好的程序员能在十分之一时间里解决给定问题。可若问题尚未给定呢？编程与许多领域一样，难的不是解题，而是决定解什么题。想象力难测，实践中却主导着以代码行数衡量的那种生产力。

### 杠杆越来越长

任何领域生产力都有差异，但很少有领域像编程差这么多——差异大到成了**种类之别**。我不认为这是编程固有的；每个领域里，技术都会放大生产力差异。编程里发生的是：我们拥有大量技术杠杆。而每个领域的杠杆都在变长，所以更多领域会陆续看到我们如今在编程里看到的那种差异。公司与国家的成败，将越来越取决于如何应对这一点。若生产力差异随技术而扩大，最高产者的贡献不仅不成比例地大，还会随时间**继续增长**。当一组人 90% 的产出由 1% 的成员创造时，若维京劫掠或中央计划之类的事把他们的生产力拉低到平均水平，损失就惨重。要充分利用他们，就得理解这些特别高产的人：什么激励他们？他们需要什么才能工作？怎么认出他们？怎么让他们为你工作？当然还有：你怎么成为他们之一？

### 不止为钱

我认识一些超级黑客，坐下来想他们有什么共同点。定义性品质大概是：**他们真的爱编程**。普通程序员写代码是为了付账单；伟大的黑客把这当作好玩的事，而且很高兴发现有人愿意为此付钱。伟大的程序员有时被说成对钱漠不关心——不完全是。他们真正在乎的是做有趣的工作；但若你赚够钱，就可以做任何想做的事，所以「赚大钱」对黑客有吸引力。只要还得每天上班，他们更在乎在那里做什么，而不是拿多少报酬。

### 爱编程的经济学

从经济学看，这是最重要的事实：**你不必按价值付给优秀黑客报酬**。伟大程序员的生产力可能是普通人的十倍或百倍，却会觉得自己很幸运能拿三倍工资——部分因为优秀黑客不知道自己有多优秀（后文再讲），也因为钱不是他们最想要的东西。

### 好工具

黑客要什么？像所有工匠一样，他们要**好工具**——这还是轻描淡写了。优秀黑客无法忍受烂工具；基础设施选错的项目，他们干脆拒绝参与。我曾供职的一家初创公司，布告栏上贴着 IBM 广告：一张 AS/400 的图片，标题大概是「黑客鄙视它」。[1]

### 语言即社区

为项目选基础设施，你不只是在作技术决策，也在作**社会决策**——后者可能更重要。比如公司要写软件，用 Java 似乎审慎；但选语言就是选社区：能雇来写 Java 项目的程序员，不会像写 Python 项目时雇到的那么聪明。黑客素质可能比语言选择更重要；不过坦白说，好黑客偏爱 Python 而非 Java，本身就该说明两种语言的相对优劣。

### 语言是表达媒介

商业人士偏爱最流行的语言，因为他们把语言当**标准**，不想把公司押在 Betamax 上。但语言不只是标准：传比特用 TCP/IP 没错，编程语言却不只是格式，而是**表达媒介**。我读到 Java 刚超过 Cobol 成为最流行语言——作为标准，不能再好了；作为表达媒介，你可以好得多。我能想到的所有伟大程序员里，自愿用 Java 的我只认识一个；不在 Sun 做 Java 的伟大程序员，我认识的是零个。

### 开源与控制

伟大黑客通常坚持用开源软件——不只因为更好，更因为**控制权**更大。好黑客坚持控制：东西坏了要能修，这是他们成为好黑客的部分原因。你希望他们对为你写的软件有这种感觉；对操作系统有同样感觉，也不该惊讶。几年前一位风投朋友告诉我他投的新公司，听起来不错；下次再聊，他说决定在 Windows NT 上构建，刚雇了一位很有经验的 NT 开发者当 CTO。我一听就想：这些人完蛋了。其一，这位 CTO 不可能是顶尖黑客——要成为杰出的 NT 开发者，得多次**自愿**使用 NT，我想象不出伟大黑客会这么做；其二，就算他不错，项目必须建在 NT 上，他也很难雇到优秀的人。[2]

### 办公室

软件之外，黑客最重要的工具大概是**办公室**。大公司认为办公空间的功能是表达职级；黑客却不止于此：办公室是**思考的地方**。你是科技公司，他们的想法就是你的产品——让黑客在嘈杂、分心的环境里工作，就像油漆厂空气里满是煤烟。《呆伯特》对小隔间有很多话，理由充分：我认识的黑客都鄙视隔间。

### 隔间与专注

仅仅**可能被中断**，就足以阻止黑客解难题。要在有隔间的办公室做真活，只有两招：在家工作，或早到、晚走、周末来——趁没别人。公司难道看不出这是出了问题的信号？办公环境本该帮你工作，而不是你**尽管**环境去工作。思科以人人有小隔间、连 CEO 也不例外而自豪——他们并不像自以为的那么先进，显然仍把办公空间当职级徽章。注意思科也以几乎不做内部产品开发闻名；新技术靠收购初创公司获得——那些黑客大概确有安静的工作场所。

### 微软与在家工作

理解黑客需求的大公司之一是微软。我见过微软招聘广告：一扇门的大图——「来为我们工作，我们会给你一个能真正干活的地方。」微软在大公司里难得之处在于能**自行开发**软件——也许不算特别好，但够用了。若公司想让黑客高产，该看看他们在家做什么：自己安排环境以最大化产出；不在嘈杂开放空间，而在**有门的房间**；在舒适、有邻里感、能散步琢磨问题的地方，而不是玻璃盒加数英亩停车场；有沙发可小憩，而不是坐桌前装忙；黄金编程时段没有吸尘器大队呼啸而过；没有会议，更没有公司务虚或团建。看他们在计算机上做什么，会印证我前面关于工具的说法：上班可能得用 Java 和 Windows，在家自选时，更可能发现 Perl 和 Linux。

### 自由选择时的工具

Cobol 或 Java「最流行」的统计可能误导。要知道什么工具最好，该看黑客**自由选择**时用啥——即自己的项目。那时你会发现开源操作系统已占主导，排名第一的语言大概是 Perl。

### 有趣的项目

除好工具外，黑客还要**有趣的项目**。什么有趣？显然隐形飞机、特效软件这类性感应用有趣；但任何应用只要提出新颖技术挑战就会有趣——所以难预测黑客会喜欢什么问题，有些问题只有研究者找到新解法后才变得有趣。ITA（Orbitz 内部软件）之前，做机票搜索的人大概觉得这是最无聊的应用之一；ITA 用更雄心勃勃的方式**重新定义问题**，把它变有趣了。谷歌大概也如此：成立时门户圈的常识是搜索无聊且不重要，谷歌的人不这么认为，所以做得好。

### 经理能重新定义问题

这是经理能发挥作用的领域。就像父母对孩子说「我赌你十分钟内收拾不完整个房间」，好经理有时能把问题重新定义成更有趣的。乔布斯似乎特别擅长，部分因为他标准高：Mac 之前已有很多小型廉价电脑，他把问题重定义为：**做一个漂亮的东西**——这可能比任何胡萝卜加大棒都更能驱动开发者。他们确实做到了：Mac 刚出现时，甚至不用开机就知道它会好，看机箱就知道。几周前我在剑桥街上走，某人的垃圾里有个像 Mac 手提箱的东西，里面是 Mac SE；我带回家插上，它就启动了——快乐的 Macintosh 脸，然后是 Finder。天哪，就这么简单。就像……谷歌。

### 高标准与品味

黑客喜欢为**高标准**的人工作；但严格要求不够，还得坚持**正确的事**——通常意味着你自己得是黑客。偶尔看到「如何管理程序员」的文章；其实该有两篇：一篇给程序员自己，一篇给不是程序员的——后者大概可浓缩为两个字：**放弃**。问题不在日常管理：真正优秀的黑客几乎自我管理。问题是你若不是黑客，就**认不出**谁是好黑客——类似的问题解释美国车为何丑，我称之为**设计悖论**。

### 设计悖论

你可能以为雇一位伟大设计师就能让产品漂亮；但若自己没好品味，怎么认出好设计师？按定义你无法从作品集看出。也不能靠奖项或履历——设计领域与多数领域一样，这些往往由时尚和应酬驱动，真实能力远居第三。绕不过去：**不懂美，就无法管理旨在产出美之物的流程**。美国车丑，因为美国汽车公司由品味差的人经营。这国许多人把品味当成捉摸不定甚至无聊的东西——两者都不是。要驱动设计，经理必须是公司产品最苛刻的用户；若你真有品味，可以像乔布斯那样，把「让你满意」变成优秀人才爱解的问题。

### 讨厌的小问题

什么类型的问题**不有趣**，相当好说：不是解几个大而清晰的问题，而是得解大量讨厌的小问题。最糟的一类之一，是为充满 bug 的软件写接口；另一类是按个别客户复杂、含糊的需求做定制。对黑客来说，这类项目是**凌迟**。讨厌小问题的显著特征是：**你从中学不到东西**。写编译器有趣，因为它教你编译器是什么；为有 bug 的软件写接口教不了你什么，因为 bug 是随机的。[3] 所以优秀黑客避开它们，不只因挑剔，更是**自我保护**——干讨厌小问题会让你变蠢，就像模特避开芝士汉堡。

### 如何让聪明人干苦活

当然有些问题天生如此，供需使报酬特别高——找到办法让伟大黑客干乏味活的公司会非常成功。怎么做？**初创公司**是一处：我们初创公司里 Robert Morris 当系统管理员——像滚石乐队在成人礼上演出，这种人才雇不到；但人们会为自己创办的公司干任何苦差。[4] 大公司靠**分割**：设独立研发部门，员工不必直接面对客户那些讨厌小问题。[5] 研发部门像矿井，产出新点子，也许其他部门能用。不必走到这步：**自下而上编程**建议另一种分法——让聪明人当工具制造者。若公司做某类软件，一组人建写这类软件的工具，另一组用工具写应用；这样也许 99% 代码仍由聪明人写，却几乎与用户隔离，像传统研发部。工具制造者有用户，但只是公司自己的开发者。[6] 若微软用这法子，软件不会满是安全漏洞——写应用的不太聪明的人不会去做分配内存这类底层活；不是直接用 C 写 Word，而是拼 Word 语言的大乐高块（Duplo，我想是技术术语）。

### 聚群

除有趣问题外，好黑客还喜欢**其他好黑客**。伟大黑客往往聚群——有时惊人地聚，如施乐帕洛阿尔托研究中心。所以你吸引到的优秀黑客，并不与你创造的环境好坏成线性比例；聚群趋势使关系更像环境的**平方**——赢家通吃。任何时刻，黑客最想工作的地方大概只有十到二十个；你若不在其中，优秀黑客不会只是少几个，而是**零**。

### 优秀黑客与公司

拥有伟大黑客本身不足以让公司成功——对谷歌和 ITA 这两个热点很好，对 Thinking Machines 或 Xerox 没帮助。Sun 曾风光一阵，商业模式却是下行电梯——那种情况下最好的黑客也救不了你。但我认为，其他条件相同时，能吸引伟大黑客的公司会有巨大优势。有人不同意：1990 年代我们跑风投时，几家告诉我们软件公司不是靠写出色软件赢，而是靠品牌、渠道和交易。

### 风投的误判

他们似乎真信这个，我想我知道为什么：很多 VC 无意识在找**下一个微软**。若微软是榜样，就不该找指望靠出色软件赢的公司。但 VC 找下一个微软是错的——除非有别的公司准备在恰当时机弯腰成为下一个 IBM，否则没有初创能成为下一个微软。以微软为榜样是错误：他们的整个文化源于那一次幸运突破——微软是**坏数据点**。扔掉它，你会发现好产品确实往往赢得市场。VC 该找的是下一个苹果，或下一个谷歌。我想比尔·盖茨知道这一点：让他担心谷歌的不是品牌力量，而是他们有**更好的黑客**。[7]

### 如何识别

那么谁是伟大黑客？遇见时怎么知道？非常难——连黑客自己也难辨。我现在相当确定朋友 Trevor Blackwell 是伟大黑客；你可能在 Slashdot 上读过他自制 Segway——值得注意的是他在**一天内**写完所有软件（顺便说，用 Python）。对 Trevor 这稀松平常；但我初见时以为他是十足白痴——站在 Robert Morris 办公室里对他胡言乱语，我在后面疯狂手势，叫 Robert 把这疯子赶走好去吃午饭。Robert 说他起初也误判 Trevor：第一次见面时 Trevor 刚开始新计划，把生活各方面一切写在随身携带的一叠索引卡上；他刚从加拿大来，浓重口音加鲻鱼头。

### 社交伪装与简历

黑客尽管以不善社交闻名，有时却花很大力气**显得聪明**，问题更复杂。读研究生时我偶尔去 MIT AI Lab 晃——起初吓人，人人说话飞快；后来我学会快说的技巧：不必想更快，用两倍词数说同样的事就行。信号里噪音这么多，遇见优秀黑客时很难辨认——即使现在我也说不清。**简历也看不出**。判断黑客似乎只能与他一起做件事——这也是高科技区域只出现在大学附近的原因。

### 大学与高-tech 集群

这里的活性成分与其说是教授，不如说是**学生**。初创公司在大学周围生长，因为大学把有前途的年轻人聚在一起，让他们做同一类项目；聪明的人认出其他聪明人，一起酝酿新项目。因为除非共事否则认不出伟大黑客，黑客自己也**不知道自己有多优秀**——多数领域某种程度上如此：擅长某事的人，与其说确信自己伟大，不如说困惑于别人为何这么无能。但对黑客尤其难，因为难比较彼此工作——百米十秒就知道谁最快；数学里对难题与好解似乎也有共识；**编程像写作**，谁能说两本小说哪本更好？作者当然不能——至少黑客之间能看出来，因为他们与小说家不同，在项目上**协作**；网上抛难题过去，很快知道对方回击多狠。但黑客无法观察自己工作——所以问伟大黑客他多优秀，几乎一定回答「不知道」；不是谦虚，是真不知道。除了实际共事过的人，我们谁也不知道——这把我们置于怪境：不知道英雄该是谁。成名的黑客往往靠公关偶然事件成名；偶尔要举伟大黑客的例子，我从不知道用谁——脑海第一个总是认识的人，用他们似乎很拿不出手；也许该说 Richard Stallman、Linus Torvalds、Alan Kay 之类名人——但我不知道他们是不是伟大黑客，从未与他们共事。若黑客界有迈克尔·乔丹，没人知道，包括他自己。

### 修炼

最后，黑客们都想知道的问题：**如何成为伟大黑客？** 我不知道能否把自己炼成；但当然可能做事让自己变蠢——若能变蠢，大概也能变聪明。成为优秀黑客的关键也许是**做你喜欢的事**。我认识的伟大黑客有个共同点：极难让他们做不想做的事——不知因果，可能两者皆是。要把事做好，你得**爱**它；所以尽量让编程保持为你所爱，你就可能做得好。尽量保留十四岁对编程的那份惊奇。若担心当前工作正在腐蚀大脑，大概确实在腐蚀。

### 好奇心

最好的黑客往往聪明——许多领域皆然。黑客有独特品质吗？我问几个朋友，第一条是**好奇心**。我一直以为聪明人都好奇——好奇只是知识的一阶导数；但黑客似乎特别好奇，尤其对**事物如何运作**——合理，因为程序本质上是对运作方式的大描述。

### 专注

几个朋友提到黑客的**专注**——用他们的话说，能「屏蔽自己脑袋以外的一切」。我确实注意到；还听过几个黑客说喝半杯啤酒就完全没法编程——也许编程确实需要某种特殊聚焦能力。也许伟大黑客能在脑中载入大量上下文，看一行代码时看到的不仅是那一行，而是周围整个程序。 John McPhee 写 Bill Bradley 篮球成功，部分因为异常的周边视野：「完美」视力垂直周边约 47 度，Bradley 有 70 度——看地板时能看到篮筐。也许伟大黑客有类似天赋。（我用非常**密集**的语言作弊，缩小了球场。）这或可解释隔间问题上的隔阂：管设施的人若没有可被击碎的专注，大概不知道在小隔间工作对黑客像把脑子放进搅拌机。（若 Bill 有自闭症的传闻为真，他倒是太清楚了。）

### 态度

我注意到伟大黑客与一般聪明人的一点差异：黑客更**政治不正确**。好黑客之间若有暗号，大概是彼此够熟，能表达会被公众石刑的观点。编程里政治不正确有用：程序很复杂，至少在好程序员手里很灵活——这种情境下，习惯质疑假设有帮助。

### 成为黑客的配方

这些品质能培养吗？我不知道；但至少可以不**压制**它们。以下是我最好的配方：若可能把自己炼成伟大黑客，办法也许是与自己立约——**永远不必做无聊项目**（除非家人会饿死），作为交换，**永远不允许自己半吊子**。我认识的伟大黑客似乎都做了这交易，也许他们别无选择。
