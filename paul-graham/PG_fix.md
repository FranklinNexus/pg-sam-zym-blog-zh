# 编程语言各自修复了什么

> 原文：<https://paulgraham.com/fix.html>
> 来源：[paulgraham.com](https://paulgraham.com/)
> 英文标题：What Languages Fix

### 用「修复了什么」来比较语言

Kevin Kelleher 提出了一种有趣的方式来比较编程语言：**用每种语言所解决的问题来描述它。**令人惊讶的是，有多少语言可以用这种方式来描述——而且描述得相当准确。

**Algol：** 汇编语言太底层了。

**Pascal：** Algol 的数据类型不够多。

**Modula：** Pascal 太弱了，做不了系统编程。

**Simula：** Algol 做模拟不够好。

**Smalltalk：** Simula 里不是所有东西都是对象。

**Fortran：** 汇编语言太底层了。

**Cobol：** Fortran 太吓人了。

**PL/1：** Fortran 的数据类型不够多。

**Ada：** 现有的每种语言都缺点什么。

**Basic：** Fortran 太吓人了。

**APL：** Fortran 处理数组不够好。

**J：** APL 需要自己的字符集。

**C：** 汇编语言太底层了。

**C++：** C 太底层了。

**Java：** C++ 太乱了。而且微软要碾压我们了。

**C#：** Java 被 Sun 控制了。

**Lisp：** 图灵机是一种笨拙的计算描述方式。

**Scheme：** MacLisp 太乱了。

**T：** Scheme 没有库。

**Common Lisp：** Lisp 的方言太多了。

**Dylan：** Scheme 没有库，而且 Lisp 语法太吓人了。

**Perl：** Shell 脚本/awk/sed 不够像编程语言。

**Python：** Perl 太乱了。

**Ruby：** Perl 太乱了，而且 Lisp 语法太吓人了。

**Prolog：** 编程不够像逻辑。
